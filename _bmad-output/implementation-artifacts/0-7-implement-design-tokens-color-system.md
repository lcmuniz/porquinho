### Como Funciona o @theme no Tailwind v4

No Tailwind v4, para que uma CSS variable se torne uma classe Tailwind utility, ela DEVE ser registrada no bloco `@theme inline`:

```css
@theme inline {
  /* Isso cria a classe bg-success, text-success, border-success, etc. */
  --color-success: var(--success);
  --color-success-foreground: var(--success-foreground);
}

:root {
  --success: oklch(0.627 0.176 149.579); /* Green 600 */
  --success-foreground: oklch(0.985 0 0); /* White */
}
```

Para cores com escala numerica (gray-50, gray-100, etc.), registrar como:
```css
@theme inline {
  --color-gray-50: var(--gray-50);
  --color-gray-100: var(--gray-100);
  /* etc. */
}
```

### WCAG 2.1 AA Contrast Requirements

Todas as combinacoes de texto/fundo devem atender:
- **Normal text (< 18px bold ou < 24px):** ratio >= 4.5:1
- **Large text (>= 18px bold ou >= 24px):** ratio >= 3:1
- **UI components:** ratio >= 3:1

Pares ja validados pela UX spec:

- Purple 600 on White: 4.75:1 (AA) ✅
- Green 600 on White: 4.54:1 (AA) ✅
- Red 600 on White: 5.14:1 (AA) ✅
